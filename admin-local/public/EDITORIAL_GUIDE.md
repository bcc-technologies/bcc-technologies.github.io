# Guía operativa editorial BCC

## Flujo recomendado para una entrada

1. Crear o abrir la entrada.
2. Completar título, fecha, sección e idioma.
3. Escribir el cuerpo del artículo.
4. Asignar autoría en metadata.
5. Añadir excerpt, tags y cover.
6. Asociar recursos recomendados si aportan valor real.
7. Insertar widgets solo cuando ayuden a entender, explorar o demostrar algo.
8. Renderizar la plantilla en desktop y móvil.
9. Revisar Salud editorial.
10. Regenerar la entrada y publicar.

## Cuándo usar widgets

Usa widgets para convertir una idea en evidencia, exploración o demostración.
No los uses como decoración.

### Widgets recomendados por caso

- Idea clave: principios, definiciones, advertencias o preguntas fuertes.
- Imagen avanzada: micrografías, diagramas, overlays, screenshots.
- Comparador: antes/después, original/segmentado, filtro A/B.
- Tabla: parámetros, resultados, mediciones, criterios.
- Gráfico: tendencias, comparaciones, curvas sencillas.
- Métricas: resultados resumidos y fáciles de escanear.
- Scrollbox: derivaciones, código, detalles metodológicos largos.
- Demo: exploración ligera con slider o toggle.
- Producto/servicio: conexión editorial con una oferta real de BCC.

## Checklist antes de publicar

- Título claro.
- Excerpt útil.
- Autoría asignada.
- Tags suficientes.
- Cover correcto.
- Widgets sin errores.
- Recursos recomendados relevantes.
- Render móvil revisado.
- Render desktop revisado.
- Artículo regenerado con plantilla actual.

## Regla editorial

Si el bloque no mejora comprensión, confianza, exploración o conversión, probablemente sobra.
